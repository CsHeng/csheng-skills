# Implementation Closeout

Use after completing edits, verification, deployment, install, or cleanup work.

## Required Content

- What changed.
- Verification run and pass/fail status.
- Anything not run or not completed.
- Follow-up only when it is directly useful and grounded in the request.

## Rules

- Do not claim completion before the requested write, install, deploy, or verification command has succeeded.
- Keep the closeout scoped to touched files and observed behavior.
- Avoid long implementation narratives.
- If memory was used and the active environment requires memory citation, include the required citation block outside generated artifacts.
